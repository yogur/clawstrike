## Codebase Patterns
- Use `StrEnum` (Python 3.11+) for all enum classes — ruff UP042 flags `(str, Enum)` style.
- Pydantic config models: `ConfigDict(extra="allow")` so unknown fields land in `model_extra` for warning logic.
- `load_config(path)` in `config.py` is the single config entry point; raises `FileNotFoundError` or `ValueError`.
- MCP server uses a module-level `mcp = FastMCP(...)` + `_config` global; `init_server(cfg)` injects config. Tests use an `autouse` fixture that resets `srv._config = None` for isolation.
- FastMCP v3 tool decorator: `@mcp.tool` (no parentheses). Test with `await mcp.call_tool(name, args)`, access result via `result.structured_content`.
- `RuntimeError` inside FastMCP tools is wrapped as `ToolError` at the protocol boundary — assert `ToolError` in tests, not `RuntimeError`.
- `fastmcp run` support: set `CLAWSTRIKE_CONFIG=<path>` env var; module auto-initializes on import.
- ruff line length: 88 chars — always use `ruff format`, never manually wrap.

- **Learnings for future iterations:**
  - FastMCP v3 is already at 3.0.1; the `@mcp.tool` decorator (no parentheses) is the current API.
  - Direct tool testing: `result = await mcp.call_tool("name", {…})` works in-process without a Client. `result.structured_content` returns the typed dict. `result.data` also available.
  - `RuntimeError` inside a tool handler is caught and re-raised as `fastmcp.exceptions.ToolError` — tests must match `ToolError`, not `RuntimeError`.
  - Module-level `mcp = FastMCP(...)` must exist for `fastmcp run server.py` to find the server; don't use a factory-only pattern if you want `fastmcp run` support.

  - `mcp.run(transport="stdio")` is the correct call; the `transport` kwarg is a string literal.
  - Startup log must go to stderr (stdout is reserved for MCP protocol messages in stdio mode).
